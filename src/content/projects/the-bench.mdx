The Bench is my fantasy basketball web app focused on making sports data feel actionable instead of overwhelming. I wanted to build something closer to a real product than a class project, so I treated it like a production system: proper schema design, authentication, data ingestion, transactional logic, and a polished frontend that someone could actually use repeatedly.

I built the app with Next.js 15, TypeScript, Supabase, Tailwind, and Vercel. On the backend/data side, I designed a normalized relational schema covering seasons, teams, players, games, and parlays, with row-level security so each user only sees their own data. I also built CSV ingestion flows for large historical datasets and made sure the pipeline preserved referential integrity across interconnected tables.

One part I am especially proud of was the transactional parlay engine. I implemented atomic balance deduction, rollback on failure, and trigger-based payouts so the app behaved consistently even when multiple actions touched the same betting or wallet state. On the frontend, I built a mobile-responsive interface with a global cart context, optimistic updates, and typed validation, which taught me a lot about keeping product UX fast without sacrificing correctness.

This project pushed me to think like both an engineer and a product builder. I learned how much full-stack quality depends on invisible systems work — schema design, auth boundaries, validation, and failure handling — not just attractive UI. It also gave me a strong sandbox for integrating predictive models into a real user-facing product rather than leaving them in notebooks.
